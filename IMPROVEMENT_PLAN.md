# diveo Improvement Plan (v2 — review-hardened)

> Produced from a verified 7-dimension maintainability audit (2026-06-20), then hardened by a 6-agent review panel (sequencing, risk/rollback, effort/scoping, completeness, technical-correctness, verifiability). Every task cites the audit finding + file:line evidence and has a **runnable** acceptance check.

## What changed from v1 (the panel's load-bearing corrections)

1. **`eas.json` is dead config for the shipped APK.** `release.yml` builds via raw `expo prebuild` + `gradlew assembleRelease`, **not** `eas build`. The released APK's env comes from `process.env` / workflow step env, never `eas.json`. ⇒ Fixing the GSAV URL in `eas.json` is a **false-green**: the verifier passes while the APK still ships `http://127.0.0.1:5191`. Acceptance must inspect the **built artifact**.
2. **Typed routes are already ON** (`app.json:49-51`). The real defect: `.expo/types` is only generated by the dev server (and is gitignored), so `tsc` never enforces routes, and `as any` casts mask it.
3. **The entire GSAV surface is untracked in git** — `app/gsav`, `app/watch`, `app/gsav-diagnostics`, `components/GsavWebView`, `utils/gsavBridge(+test)`, `scripts/gsav-*`, the verifier(+test), `docs/`, plus modified `app/_layout.tsx`/`app/index.tsx`. A fresh clone has **zero tests**. This is now the first commit.
4. **`.env` is already tracked**; `.gitignore` doesn't cover bare `.env`. Untrack it; the Sentry **DSN is public-by-design** and needs no rotation — only the **auth token** does.
5. **`docs/*` is gitignored** (`.gitignore:48`) ⇒ an ADR under `docs/` is silently un-committable. Fix gitignore or relocate.
6. **P3-3 must not build a catalog** — the checklist (`:16,68-69,175`) assigns catalog ownership to `../gsav-hosting`. The home screen is a thin **launcher**, not a catalog.

## Guiding principle

~80% of audit findings trace to the **unmanaged legacy↔GSAV pivot**. Resolving it is the keystone: most debt either *disappears* (delete legacy) or becomes *tractable* (a small bounded shell). **Corollary rule:** do not invest medium/large effort in legacy code that has a deletion date — freeze it, then delete it.

## ⚠️ Keystone decision gate (DG-1) — decide in Phase 2

| Option | Meaning | Verdict |
|---|---|---|
| **A — Freeze & quarantine (RECOMMENDED)** | Legacy reachable behind a flag, no new investment, **concrete deletion date** | Safest path to GSAV-as-product |
| B — Delete legacy | Remove Bilibili surface entirely (see **Phase 4B/5B**) | Max debt reduction; only AFTER GSAV is a usable product |
| C — Keep both | Status quo | Rejected — this generated the audit |

**DG-1 output is not just "A/B/C" — it must set a literal legacy-deletion date** (e.g. "delete by v1.1.0 / 2026-08-31"). Any legacy-only refactor whose payoff horizon exceeds that date is **DROPPED**. DG-1 done = `docs/adr/0001-gsav-pivot.md` exists with `Status: Accepted`, the chosen option, and the date; Phase 4/5 notes reference the option by ID.
